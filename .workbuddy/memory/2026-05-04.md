# PomodoroX 开发日志

## 2026-05-04 第四轮：Outbox 事件流架构（同步模型重构）

### 基于 Codex 深度审查的架构重构

**核心改动**：将 GitHub 同步从"覆盖 JSON 快照"模式改为"append-only 事件流"模式

### 新增模块

| 文件 | 职责 |
|------|------|
| `src/services/outbox.ts` | Append-only 事件队列。每条 CRUD 生成一个不可变事件（eventId + type + payload），本地 IndexedDB 持久化 |
| `src/services/event-consumer.ts` | 幂等事件消费者。从 GitHub 拉取 outbox 事件 → 按 eventId 去重 → 逐一写入本地 |

### 修改模块

| 文件 | 改动 |
|------|------|
| `src/services/github.ts` | 新增 `pushEvent()` / `pushEvents()` / `pullEvents()` — 推拉 outbox 事件（Append-only，永不覆盖已有文件） |
| `src/stores/sync.ts` | 全面重写：串行锁（serialized）、recordEvent()、backgroundPull()、outbox-based sync |
| `src/stores/task.ts` | CRUD 后调用 `recordEvent('task.created/updated/deleted', ...)` 写入 outbox 事件 |
| `src/stores/reflection.ts` | 同上（reflection.created/updated/deleted） |
| `src/stores/session.ts` | 同上（session.created） |
| `src/App.vue` | 启动时调用 `backgroundPull()` 消费远程事件 |

### 架构对比

```
之前（JSON 快照覆盖）:
  手机写 → 每月 tasks.json（覆盖）→ GitHub
  桌面写 → 每月 tasks.json（覆盖）→ 冲突

之后（Append-only 事件流）:
  手机写 → outbox/{eventId}.json（新建，永不覆盖）→ GitHub
  桌面写 → outbox/{eventId}.json（新建，永不覆盖）→ GitHub
  任何一端启动 → 拉取所有 outbox/*.json → 按 eventId 去重 → 幂等写入本地
  删除 → task.deleted 事件（tombstone 跨端可见）
```

### 关键质量属性

- **幂等消费**：同一 eventId 多次拉取只处理一次
- **串行安全**：全局 serialized 锁，防止并发同步冲突
- **无覆盖风险**：Append-only 模式，每个事件的 JSON 文件名带 eventId，永不覆盖
- **删除可传播**：delete 事件携带实体 ID，消费端执行本地删除
- **限流安全**：推拉各限制 10/100 个/批，避免 API 限频

### Vercel 部署修复

**问题**：`vercel --yes` 首次部署时自动创建了随机名项目 `dist-web`，而非用户已有的 `pomodorox-web`。`pomodorox-web` 显示 "No Production Deployment"。

**根因**：无 `.vercelignore` → CLI 尝试上传整个项目（含 `src-tauri/target/` 4.2GB）→ 部署失败 → CLI 回退创建新项目 `dist-web`。

**修复**：
- 创建 `.vercelignore`：排除 `node_modules/`, `src-tauri/target/`, `.git/`, `src/`, `scripts/` 等
- 创建 `vercel.json`：`{"buildCommand": null, "outputDirectory": "dist-web", "framework": null}` → 直接部署预构建产物，跳过云端构建
- 重新执行 `vercel --prod` → 成功部署到 `pomodorox-web`
